Secret Sharing &
Threshold Cryptography

A distributed key management system where a cryptographic secret can only be recovered when a minimum number of trusted participants collaborate.

How It Works
1

Admin generates a secret

A random AES-256 key is created as the master secret.

2

Split into n shares

Shamir's scheme splits the secret into n mathematical shares for executives.

3

k-of-n threshold

Any k shares reconstruct the secret. Fewer than k reveals nothing.

4

Lagrange interpolation

k points uniquely define the polynomial whose y-intercept is the secret.

Attack Simulations
A1

Insufficient Shares

Submit fewer than k shares — threshold check blocks reconstruction.

A2

Share Tampering

Alter a share's value — HMAC mismatch catches it before interpolation.

A3

Brute Force

Search space is p^(k−1) with 256-bit prime — computationally impossible.

A4

Insider Collusion

Duplicate x-values and HMAC checks block colluding executives.
